# 观星问易 · 产品说明

> 一个东西方结合的情绪陪伴产品：西方占星（天象/月相）× 东方易经卦象 × AI 伙伴「小易」。

## 1. 定位

观星问易不是算命工具，而是**借象征语言做情绪陪伴与自我整理**的产品。占星与卦象在这里是「镜子」，帮用户看见此刻的情绪与重心；小易是温和克制的陪伴者，不是客服，也不是预言家。

**目标用户**：处在情绪波动期、希望被温和地倾听和引导整理思绪的人；对东方易学/西方占星有兴趣但不迷信的年轻用户。

## 2. 安全边界（产品红线）

- **不预测命运**：所有解读落在「此刻的重心/功课」，不给「结局」。
- **不制造恐惧**：无凶卦恐吓、无负面断言；文案永远给出口。
- **不诱导依赖**：鼓励用户回到自己的生活，行动建议均是「最小的一步」。
- **不做医疗诊断**：报告与解读均带免责声明，不替代专业心理/医疗服务。

文案基调：温和、克制、留白；多用「先……就好」「不必急着……」句式。

## 3. 功能模块

| 模块 | 路由 | 说明 |
|---|---|---|
| 今日 | `/app` | 工作台首页：今日星象/月相、今日一卦、最近问卦、近 7 天情绪轨迹、小易入口、报告入口 |
| 问卦 | `/app/ask` | 完整仪式流：输入困扰 → 小易改写成 2-3 个「我可以如何」式问题 → 选择 → 按住摇签起卦 → 本卦/变爻/之卦 + 象/译/行三层解读 → 保存 |
| 小易 | `/app/chat` | 陪伴式聊天，支持语音输入；语气温和不下判断 |
| 心境 | `/app/journal` | 情绪命名（平静/疲惫/烦躁/低落/期待/紧绷）+ 紧绷程度(0-10) + 今日一件小事；历史记录与七天轨迹 |
| 姻缘 | `/app/love` | 双人信息 + 双盘交叠图 + 关系卦 + 三段分析（吸引点/需要照顾的地方/沟通建议）；不打分、不预测结局 |
| 深度报告 | `/app/report` | 按月聚合的长文报告：星盘分析/卦象分析/情绪主题/关系建议/今日行动/反思问题 六段 |

另有：落地页 `/`（产品介绍）、设计系统页 `/app/system`（内部用）。

## 4. 当前阶段（2026-07）

**mock 闭环阶段已完成**：前后端 11 个接口全部打通并端到端验证。

- 前端：Next.js 14 一套响应式代码覆盖移动端 + 桌面宽屏。
- 后端：Spring Boot 3 + PostgreSQL + Redis，全部业务数据真实落库。
- **AI 内容目前为 mock**（固定文案/固定卦象），语音转写也是 mock 文本；架构上已留好切换真实 Claude / 真实 STT 的开关（见 `docs/backend-plan.md` 第 7 节）。
- 后端不可用时前端自动回退到本地静态内容，页面不会白屏或报错。

## 5. 路线图（未排期）

1. **接真实 Claude AI**（`gxwy.ai.provider=claude`）：问题改写、解卦、聊天、姻缘分析、报告生成全部真实化——产品价值核心。
2. 真实语音识别（独立 STT，如 Whisper；路径 `/api/voice/transcriptions` 已预留）。
3. 真实起卦算法与占星计算（当前卦象/星象为固定演示内容）。
4. 姻缘页「编辑两人信息」真实表单；报告导出 PDF/分享。
5. 鉴权与多用户（当前用 `X-User-Id` 头模拟身份）。
6. 移动端原生级流程细化（摇签全屏、语音长按说话、权限弹层）。
